# 点赞模块设计文档

## 1. 模块目标

点赞模块用于为文章提供最基础的轻量互动能力，使用户无需发表评论也能表达反馈。

本模块首版目标包括：
- 已登录用户对文章点赞
- 已登录用户取消点赞
- 文章详情页显示当前点赞数
- 文章详情页显示当前用户是否已点赞
- 文章 `likeCount` 自动维护

## 2. 设计范围

### 2.1 后端范围

- 点赞关系映射
- 点赞接口
- 取消点赞接口
- 当前用户点赞状态查询
- 文章 `likeCount` 联动更新

### 2.2 前端范围

- 文章详情页点赞按钮
- 点赞状态展示
- 点赞与取消点赞交互
- 未登录点赞拦截提示

## 3. 核心设计

### 3.1 点赞关系设计

点赞使用独立关系表 `post_likes` 表示：
- 一条记录表示一个用户对一篇文章点过赞
- 通过 `(post_id, user_id)` 保证同一用户不会重复点赞同一文章

### 3.2 点赞数维护策略

- 点赞成功后，文章 `likeCount + 1`
- 取消点赞成功后，文章 `likeCount - 1`
- 不允许 `likeCount` 变成负数

### 3.3 点赞状态规则

- 未登录用户默认视为未点赞
- 已登录用户进入文章详情页时，可以查询自己是否已点赞
- 前端根据状态切换“点赞 / 取消点赞”按钮文案

## 4. API 设计

### 4.1 点赞文章

`POST /api/posts/{postId}/like`

处理逻辑：
- 仅允许已登录用户调用
- 校验文章存在且可见
- 若当前用户尚未点赞，则创建点赞关系
- 同步增加文章 `likeCount`
- 返回最新点赞状态

### 4.2 取消点赞

`DELETE /api/posts/{postId}/like`

处理逻辑：
- 仅允许已登录用户调用
- 校验文章存在且可见
- 若当前用户已点赞，则删除点赞关系
- 同步减少文章 `likeCount`
- 返回最新点赞状态

### 4.3 查询当前用户点赞状态

`GET /api/posts/{postId}/like`

处理逻辑：
- 若未登录，直接返回未点赞状态
- 若已登录，则返回当前用户对该文章的点赞状态

## 5. 返回结构设计

点赞相关接口统一返回：
- `liked`：当前用户是否已点赞
- `likeCount`：当前文章最新点赞数

这样前端在一次请求后就能同步刷新按钮状态和统计数。

## 6. 数据结构设计

### 6.1 后端重点对象

- `PostLike`
- `PostLikeRepository`
- `LikeResponse`

### 6.2 前端重点对象

- `LikeState`

## 7. 前端交互设计

### 7.1 文章详情页点赞区

页面加载时：
- 拉取文章详情
- 同步拉取当前用户点赞状态

### 7.2 点赞流程

- 若当前未点赞，点击按钮发起点赞请求
- 成功后按钮切换为“取消点赞”
- 点赞数即时更新

### 7.3 取消点赞流程

- 若当前已点赞，点击按钮发起取消点赞请求
- 成功后按钮切换为“点赞”
- 点赞数即时更新

### 7.4 未登录行为

- 未登录用户点击点赞时，提示先登录
- 不发送成功点赞请求

## 8. 安全与边界

- 点赞与取消点赞接口必须登录
- 只能对 `PUBLISHED` 文章点赞
- 重复点赞不会重复增加统计数
- 重复取消点赞不会继续减少统计数

## 9. 重点实现约束

- 文档与重点注释统一使用简体中文
- 首版只做文章点赞，不扩展到评论点赞
- 先在文章详情页完成交互，列表页点赞按钮后置

## 10. 验证方案

### 10.1 后端验证

- 未登录点赞失败
- 已登录点赞成功
- 已点赞后再次点赞不会重复增加 `likeCount`
- 取消点赞成功
- 取消点赞后 `likeCount` 正确减少

### 10.2 前端验证

- 文章详情页能显示点赞按钮和点赞数
- 点赞后按钮状态切换正确
- 取消点赞后按钮状态切换正确
- 未登录时点击点赞会提示先登录

### 10.3 联调验证

- 点赞与取消点赞后详情页统计数字立即更新
- 当前用户点赞状态与按钮状态保持一致
