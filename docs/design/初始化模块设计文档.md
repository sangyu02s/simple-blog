# 初始化模块设计文档

## 1. 模块目标

初始化模块的目标是为 Simple Blog 项目建立一个可持续扩展的前后端分离工程骨架，确保后续认证、文章、评论、点赞、后台管理等功能可以在稳定结构上逐步实现。

本阶段不追求功能完整，重点是完成以下内容：
- 后端 Spring Boot 工程初始化
- 前端 React 工程初始化
- 数据库迁移脚本初始化
- 基础页面骨架初始化
- 本地联调所需的最小配置初始化

## 2. 技术选型

### 2.1 后端

后端采用 Spring Boot 3.x，原因如下：
- 适合 Java 生态下快速搭建 Web 应用
- 与 Spring Security、Spring Data JPA、Validation 集成自然
- 便于后续扩展认证、权限和后台管理能力

当前依赖说明：
- Spring Web：提供 REST API 能力
- Spring Data JPA：支撑实体建模和数据访问
- Spring Security：为登录鉴权和角色控制做准备
- Validation：用于请求参数边界校验
- Flyway：管理数据库结构迁移
- JWT：为前后端分离登录态提供令牌支持
- H2 / MySQL Driver：兼顾本地快速开发与后续 MySQL 落地

### 2.2 前端

前端采用 React + TypeScript + Vite，原因如下：
- 启动速度快，适合从零搭建 MVP
- TypeScript 可以在项目早期就约束页面、状态和接口数据结构，降低后续联调成本
- 组件化适合博客社区页面逐步扩展
- React Router 便于管理登录、发帖、后台等页面路由
- Zustand 预留为轻量状态管理方案

## 4. 当前初始化范围

### 4.1 后端初始化内容

已完成以下基础准备：
- Spring Boot 启动类
- Maven 构建脚本
- `application.yml` 基础配置
- Flyway 初始建表脚本

当前数据库初始化表包括：
- `users`
- `posts`
- `comments`
- `tags`
- `post_tags`
- `post_likes`

这些表覆盖了博客社区首版最核心的用户、文章、评论、标签、点赞数据关系。

### 4.2 前端初始化内容

已完成以下基础准备：
- React 应用入口
- 路由骨架
- 首页、登录、注册、发帖、后台页面骨架
- 基础样式文件

当前前端页面以静态骨架为主，目的是先固定页面结构与导航流向，后续逐步接入真实 API。

## 5. 设计约束

### 5.1 模块分层约束

后端后续开发将遵循按业务拆分模块的方式，如：
- `auth`
- `post`
- `comment`
- `tag`
- `admin`

这样做可以避免早期把所有逻辑堆在单一包中，便于后续维护。

### 5.2 数据约束

初始化建表阶段已经保留以下扩展点：
- 用户 `role` 与 `status`
- 文章 `status`、`like_count`、`comment_count`、`view_count`
- 评论 `status`

这样后续做后台管理和内容可见性控制时，无需重构表结构。

### 5.3 文档与注释约束

按照项目要求：
- 设计文档始终使用简体中文
- 代码中的重点代码块需要添加简体中文注释
- 普通样板代码不强行加注释，避免噪音

## 6. 后续实施建议

初始化完成后，建议按以下顺序继续推进：

1. 认证模块
   - 用户实体
   - 注册/登录接口
   - JWT 生成与解析
   - Spring Security 鉴权配置

2. 文章模块
   - 发帖接口
   - 文章列表与详情接口
   - 标签关联逻辑

3. 评论与点赞模块
   - 评论新增与列表
   - 点赞/取消点赞
   - 计数字段维护

4. 后台管理模块
   - 管理员权限控制
   - 用户、文章、评论状态管理

## 7. 验证方案

初始化阶段的验证目标是确认工程可启动、结构合理、后续可接入功能。

### 7.1 后端验证

- 执行 Maven 构建
- 启动 Spring Boot
- 确认 Flyway 成功执行初始脚本
- 确认应用能连接内存数据库正常启动

### 7.2 前端验证

- 安装依赖并启动 Vite 开发服务
- 验证首页、登录、注册、发帖、后台页面可访问
- 验证路由跳转正常

### 7.3 联调验证

- 确认前端开发端口和后端允许跨域配置一致
- 为后续 API 对接预留统一请求入口

## 8. 总结

初始化模块的职责不是直接完成博客功能，而是先把后续开发的“地基”搭稳。当前方案已经满足从零开始建设 Simple Blog 项目的最小工程条件，接下来可以直接进入认证模块开发。
