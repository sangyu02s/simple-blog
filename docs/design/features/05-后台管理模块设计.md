# 后台管理模块设计文档

## 1. 目标

当前项目已经具备认证、发帖、评论、点赞等用户侧基础能力，但还缺少对内容和用户状态的治理能力。后台管理模块的目标是让系统从“可用”提升到“可控”，使管理员能够对文章、评论和用户进行基础管理。

本模块当前阶段目标包括：
- 管理员身份鉴别
- 文章列表管理
- 隐藏文章
- 恢复文章
- 评论列表管理
- 隐藏评论
- 恢复评论
- 用户列表管理
- 禁用用户
- 恢复用户

## 2. 范围

### 2.1 本次实现范围
- 管理员接口权限控制
- 后台文章列表接口
- 文章状态切换接口
- 后台评论列表接口
- 评论状态切换接口
- 后台用户列表接口
- 用户状态切换接口
- 前端后台文章管理页面
- 前端后台评论管理页面
- 前端后台用户管理页面

### 2.2 后续规划范围
- 后台首页统计信息

## 3. 核心设计

### 3.1 管理员权限模型

沿用现有用户角色字段：
- `USER`
- `ADMIN`

本模块约束：
- `/api/admin/**` 接口必须要求 `ADMIN`
- 普通用户访问后台接口应直接被拒绝

### 3.2 文章状态模型

沿用现有文章状态：
- `PUBLISHED`：可见
- `HIDDEN`：隐藏

状态规则：
- 前台首页与详情页只展示 `PUBLISHED` 文章
- 后台管理页可以看到所有文章
- 管理员可在 `PUBLISHED` 和 `HIDDEN` 之间切换

### 3.3 评论状态模型

沿用现有评论状态，但当前前台只展示：
- `VISIBLE`：可见
- `HIDDEN`：隐藏

状态规则：
- 前台详情页只展示 `VISIBLE` 评论
- 后台评论页可以看到所有评论
- 管理员可在 `VISIBLE` 和 `HIDDEN` 之间切换

### 3.4 用户状态模型

沿用现有用户状态：
- `ACTIVE`：可正常登录和互动
- `DISABLED`：禁止登录和继续互动

状态规则：
- 被禁用用户不能登录
- 被禁用用户即使已有 token，后续访问受保护接口时也应失效
- 后台用户页可以看到所有用户
- 管理员可在 `ACTIVE` 和 `DISABLED` 之间切换

### 3.5 后台列表展示策略

后台文章列表建议展示：
- 文章 ID
- 标题
- 作者
- 当前状态
- 创建时间
- 点赞数
- 评论数

后台评论列表建议展示：
- 评论 ID
- 所属文章标题
- 评论作者
- 评论正文摘要
- 当前状态
- 创建时间

后台用户列表建议展示：
- 用户 ID
- 用户名
- 昵称
- 角色
- 当前状态
- 创建时间

这样能支撑最基本的内容审核和状态控制。

## 4. API 设计

### 4.1 获取后台文章列表

`GET /api/admin/posts`

处理逻辑：
- 仅管理员可访问
- 返回所有文章，不区分前台可见性
- 按创建时间倒序排列

### 4.2 更新文章状态

`PATCH /api/admin/posts/{id}/status`

请求体：
- `status`

处理逻辑：
- 仅管理员可访问
- 校验文章存在
- 更新文章状态
- 返回更新后的文章摘要信息或状态信息

### 4.3 获取后台评论列表

`GET /api/admin/comments`

处理逻辑：
- 仅管理员可访问
- 返回所有评论，不区分前台可见性
- 按创建时间倒序排列

### 4.4 更新评论状态

请求体：

处理逻辑：
- 仅管理员可访问
- 校验评论存在
- 更新评论状态
- 返回更新后的评论摘要信息或状态信息

### 4.5 获取后台用户列表

`GET /api/admin/users`

处理逻辑：
- 仅管理员可访问
- 返回所有用户
- 按创建时间倒序排列

请求体：

处理逻辑：
- 仅管理员可访问

## 5. 前端交互设计

### 5.1 后台文章管理页

页面展示：
- 文章列表
- 每行一篇文章
- 显示标题、作者、状态、统计字段
- 提供隐藏/恢复按钮

### 5.2 后台评论管理页

页面展示：
- 评论列表
- 每行一条评论
- 显示所属文章、作者、评论摘要、状态
- 提供隐藏/恢复按钮

### 5.3 后台用户管理页

页面展示：
- 用户列表
- 每行一个用户
- 显示用户名、昵称、角色、状态
- 提供禁用/恢复按钮

### 5.4 权限访问行为

- 若未登录访问后台页，应请求失败
- 若普通用户访问后台页，应提示无权限或显示失败信息
- 若管理员访问后台页，应能看到列表并执行状态切换

## 8. 验证方案

### 8.1 后端验证
- 普通用户访问后台文章接口失败
- 管理员访问后台文章接口成功
- 管理员可将文章状态从 `PUBLISHED` 改为 `HIDDEN`
- 管理员可将文章状态从 `HIDDEN` 改回 `PUBLISHED`
- 普通用户访问后台评论接口失败
- 管理员访问后台评论接口成功
- 管理员可将评论状态从 `VISIBLE` 改为 `HIDDEN`
- 管理员可将评论状态从 `HIDDEN` 改回 `VISIBLE`
- 普通用户访问后台用户接口失败
- 管理员访问后台用户接口成功
- 管理员可将用户状态从 `ACTIVE` 改为 `DISABLED`
- 被禁用用户登录失败

### 8.2 前端验证
- 后台页能展示真实文章列表
- 后台页能展示真实评论列表
- 后台页能展示真实用户列表
- 管理员点击状态切换按钮后列表状态更新
- 被隐藏文章在前台首页不可见
- 被隐藏评论在前台详情页不可见
- 被禁用用户无法继续登录

## 9. 结论

后台管理模块当前阶段实现文章、评论、用户三类状态管理后，系统将具备完整的基础治理能力，后续可继续向统计视图、审核流程和更细粒度权限控制扩展。
