# 文章模块设计文档

## 1. 模块目标

文章模块用于建立博客社区最核心的内容发布与浏览能力，是后续评论、点赞、标签聚合和后台管理的基础。

本模块首版目标包括：
- 已登录用户发布文章
- 首页查看文章列表
- 查看文章详情
- 文章与标签关联
- 控制文章可见状态

## 2. 设计范围

### 2.1 后端范围

- 文章实体映射
- 标签实体映射
- 文章与标签关系映射
- 发帖接口
- 文章列表接口
- 文章详情接口
- 文章状态控制基础能力

### 2.2 前端范围

- 发帖页接入真实接口
- 首页接入真实文章列表
- 文章详情页接入真实接口
- 基础标签展示
- 发帖成功后的页面跳转

## 3. 核心设计

### 3.1 文章状态设计

首版文章状态：
- `PUBLISHED`：已发布
- `HIDDEN`：隐藏

当前不实现草稿流，优先保证社区主流程尽快可用。

### 3.2 标签处理策略

发帖时前端提交标签字符串数组。

后端处理策略：
- 去掉前后空格
- 去重
- 忽略空标签
- 若标签不存在则自动创建
- 若标签已存在则直接复用

### 3.3 内容可见性规则

- 首页列表只返回 `PUBLISHED` 文章
- 文章详情只允许查看 `PUBLISHED` 文章
- `HIDDEN` 文章为后续后台管理预留

### 3.4 统计字段策略

当前首版先维护以下字段的初始化：
- `likeCount`
- `commentCount`
- `viewCount`

发帖时默认初始化为 0。

本阶段先不实现浏览量递增逻辑，避免在详情接口阶段引入额外副作用。

## 4. API 设计

### 4.1 发布文章

`POST /api/posts`

请求体：
- `title`
- `summary`
- `content`
- `tags`

处理逻辑：
- 仅允许已登录用户调用
- 校验标题、摘要、正文长度
- 创建文章并关联作者
- 解析标签并建立关联关系
- 默认状态写为 `PUBLISHED`
- 返回文章详情结构

### 4.2 文章列表

`GET /api/posts`

处理逻辑：
- 返回所有 `PUBLISHED` 文章
- 按创建时间倒序排序
- 返回首页所需的基础字段

### 4.3 文章详情

`GET /api/posts/{id}`

处理逻辑：
- 仅允许读取 `PUBLISHED` 文章
- 返回文章详情与标签信息
- 返回作者基础信息

## 7. 前端交互设计

### 7.1 首页文章列表

- 页面加载时请求文章列表
- 展示标题、摘要、作者、标签
- 点击文章进入详情页

### 7.2 发帖页

- 已登录用户填写标题、摘要、标签、正文
- 提交后调用发帖接口
- 成功后跳转到新文章详情页

### 7.3 文章详情页

- 根据路由参数加载文章详情
- 展示标题、摘要、正文、作者、标签与统计字段

## 8. 安全与边界

- 发帖接口必须登录
- 未登录访问发帖接口直接拒绝
- 发帖人从认证上下文获取，不允许前端传作者 ID
- 文章标题、摘要、正文在接口边界做长度校验
- 标签数量和单个标签长度做基础限制，避免异常输入

## 9. 重点实现约束

- 重点代码块使用简体中文注释
- 设计文档持续使用简体中文
- 首版不做复杂分页与搜索，先跑通主流程
- 首版不引入富文本编辑器，正文先使用纯文本输入
- 文章列表和详情优先保证结构清晰，便于后续扩展评论与点赞

## 10. 验证方案

### 10.1 后端验证

- 未登录发帖失败
- 已登录发帖成功
- 发帖后标签能正确创建与关联
- 文章列表能返回新发布文章
- 文章详情能正确返回正文与标签

### 10.2 前端验证

- 首页能展示真实文章列表
- 发帖页提交后可跳转到文章详情
- 文章详情页可展示真实后端数据
- 未登录情况下发帖失败时有明确反馈

### 10.3 联调验证

- 前端发帖请求正确携带 token
- 后端能从 token 中识别作者
- 列表页和详情页返回结构与前端类型定义一致
