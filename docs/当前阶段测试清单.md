# 当前阶段测试清单

## 1. 说明

本文档用于整理 Simple Blog 当前阶段建议执行的最小手工测试清单。

当前适合重点验证的模块包括：
- 认证模块
- 文章模块

当前测试目标不是完整验收，而是确认“前后端主流程已打通，基础功能可正常运行”。

## 2. 测试前准备

在执行测试前，请先完成以下步骤：

### 2.1 启动后端

```bash
mvn -f backend/pom.xml spring-boot:run
```

### 2.2 启动前端

```bash
npm run dev --prefix frontend
```

### 2.3 打开前端页面

例如：

```text
http://127.0.0.1:5173
```

如果端口变化，请以前端启动日志中的实际地址为准。

## 3. 认证模块测试清单

### 3.1 注册新用户

#### 操作步骤
1. 打开前端首页
2. 点击“注册”
3. 输入用户名、昵称、密码
4. 提交注册表单

#### 预期结果
- 注册成功
- 页面跳转回首页
- 页面进入登录态
- 页面头部出现当前用户退出入口

---

### 3.2 重复用户名注册

#### 操作步骤
1. 使用刚才已经注册过的用户名再次注册
2. 提交注册表单

#### 预期结果
- 注册失败
- 页面显示错误提示
- 不进入新的登录态

---

### 3.3 正常登录

#### 操作步骤
1. 如果当前已登录，先退出
2. 点击“登录”
3. 输入正确用户名和密码
4. 提交登录表单

#### 预期结果
- 登录成功
- 页面跳转回首页
- 页面头部显示当前用户信息或退出入口

---

### 3.4 错误密码登录

#### 操作步骤
1. 输入正确用户名
2. 输入错误密码
3. 提交登录表单

#### 预期结果
- 登录失败
- 页面显示错误提示
- 不进入登录态

---

### 3.5 刷新后恢复登录态

#### 操作步骤
1. 先保持登录状态
2. 在浏览器中刷新当前页面

#### 预期结果
- 页面能恢复当前登录用户
- 不需要再次手动登录

---

### 3.6 退出登录

#### 操作步骤
1. 点击页面头部的退出按钮

#### 预期结果
- 本地登录态清除
- 页面不再显示当前用户退出入口
- 后续需要重新登录才能执行发帖操作

## 4. 文章模块测试清单

### 4.1 未登录时尝试发帖

#### 操作步骤
1. 确认当前处于未登录状态
2. 打开“发帖”页面
3. 填写标题、摘要、标签、正文
4. 提交表单

#### 预期结果
- 提交失败
- 页面提示“请先登录后再发布文章”或等价提示
- 不会创建新文章

---

### 4.2 登录后发布文章

#### 操作步骤
1. 先登录
2. 打开发帖页
3. 输入以下示例内容：
   - 标题：任意有效标题
   - 摘要：长度大于 10 个字符
   - 标签：多个标签用逗号分隔
   - 正文：长度大于 20 个字符
4. 提交表单

#### 预期结果
- 发帖成功
- 页面跳转到新文章详情页
- 详情页展示刚刚提交的文章内容

---

### 4.3 首页显示文章列表

#### 操作步骤
1. 回到首页

#### 预期结果
- 首页出现刚刚发布的文章
- 列表中可看到：
  - 标题
  - 摘要
  - 作者昵称
  - 标签
  - 点赞数
  - 评论数

---

### 4.4 进入文章详情页

#### 操作步骤
1. 在首页点击任意文章标题

#### 预期结果
- 成功进入文章详情页
- 页面展示：
  - 标题
  - 摘要
  - 正文
  - 作者信息
  - 标签
  - 点赞数
  - 评论数
  - 浏览数

---

### 4.5 多标签文章验证

#### 操作步骤
1. 发布一篇带多个标签的文章
2. 打开首页与详情页查看标签展示

#### 预期结果
- 标签正常展示
- 标签之间没有空值
- 标签重复输入时后端不会生成重复标签关联

## 5. 建议补充验证

如果时间允许，建议额外验证以下边界情况。

### 5.1 注册输入边界
- 用户名过短
- 昵称过短
- 密码过短

#### 预期结果
- 页面提交失败
- 后端返回参数错误提示

### 5.2 发帖输入边界
- 标题为空
- 摘要过短
- 正文过短
- 标签为空

#### 预期结果
- 提交失败
- 页面显示失败提示

### 5.3 重启后端后的数据行为

#### 操作步骤
1. 注册用户并发布文章
2. 停止后端
3. 重新启动后端
4. 再次访问前端

#### 预期结果
- 因为当前使用 H2 内存数据库，之前的数据会丢失
- 这是当前阶段的正常现象

## 6. 当前不在重点测试范围内的功能

以下内容暂时不建议作为当前阶段重点测试项，因为尚未实现或尚未完善：
- 评论功能
- 点赞功能
- 标签聚合页
- 后台管理真实功能
- 管理员权限场景
- 更复杂的 token 失效与刷新机制

## 7. 本阶段通过标准

如果以下主流程全部通过，则可以认为当前阶段开发结果基本可用：

1. 能注册新用户
2. 能登录和恢复登录态
3. 能退出登录
4. 登录后能成功发帖
5. 首页能看到已发布文章
6. 能进入文章详情页并看到正确内容

## 8. 结论

当前阶段的最小测试清单聚焦在“认证 + 发帖 + 浏览文章”闭环。只要这条主流程稳定，就说明当前项目已经具备继续扩展评论、点赞和后台管理等模块的基础。
