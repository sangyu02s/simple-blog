# 项目运行说明

## 1. 说明

本文档用于说明 Simple Blog 项目当前阶段的运行方式。

当前项目采用前后端分离结构：
- 前端：React + TypeScript + Vite
- 后端：Java 21 + Spring Boot + Maven
- 数据库：H2 内存数据库

当前已经可以运行并做简单联调测试的功能包括：
- 用户注册
- 用户登录
- 获取当前用户信息
- 发布文章
- 文章列表
- 文章详情

## 2. 环境要求

在本地运行项目前，请确保具备以下环境：

### 2.1 后端环境

- Java 21
- Maven 3.9+

### 2.2 前端环境

- Node.js 24+
- npm 11+

## 3. 项目目录结构

```text
simple-blog/
├── backend/
├── frontend/
└── docs/
```

## 4. 后端运行方式

### 4.1 启动命令

在项目根目录执行：

```bash
mvn -f backend/pom.xml spring-boot:run
```

### 4.2 默认访问地址

后端默认运行在：

```text
http://127.0.0.1:8080
```

### 4.3 当前数据库说明

当前后端默认使用：
- H2 内存数据库

特点：
- 启动快
- 适合本地开发与验证
- 服务停止后数据不会保留

### 4.4 数据库初始化

后端启动时会自动执行 Flyway migration：
- 初始化表结构
- 创建用户、文章、评论、标签、点赞等基础表

## 5. 前端运行方式

### 5.1 安装依赖

首次运行前，请执行：

```bash
npm install --prefix frontend
```

### 5.2 启动命令

在项目根目录执行：

```bash
npm run dev --prefix frontend
```

### 5.3 默认访问地址

前端通常运行在：

```text
http://127.0.0.1:5173
```

如果 `5173` 端口被占用，Vite 会自动切换到其他端口，例如：
- `5174`
- `5175`

请以前端启动日志中的实际地址为准。

## 6. 推荐启动顺序

推荐按下面顺序启动：

### 第一步：启动后端

```bash
mvn -f backend/pom.xml spring-boot:run
```

### 第二步：启动前端

```bash
npm run dev --prefix frontend
```

### 第三步：打开浏览器访问前端地址

例如：

```text
http://127.0.0.1:5173
```

## 7. 构建与测试命令

### 7.1 后端测试

```bash
mvn -f backend/pom.xml test
```

### 7.2 后端打包

```bash
mvn -f backend/pom.xml package
```

### 7.3 前端构建

```bash
npm run build --prefix frontend
```

## 8. 当前可验证功能

当前阶段建议按下面方式做最小手工验证。

### 8.1 认证模块验证

可验证内容：
- 注册新用户
- 使用新用户登录
- 刷新页面后恢复登录状态
- 退出登录

### 8.2 文章模块验证

可验证内容：
- 登录后进入发帖页
- 输入标题、摘要、标签、正文后提交
- 发布成功后跳转到文章详情页
- 首页可看到新发布文章
- 点击文章可进入详情页

## 9. 当前限制说明

当前项目还处于基础开发阶段，以下能力尚未完成：
- 评论功能
- 点赞功能
- 标签聚合页
- 后台管理真实能力
- 更完整的权限控制
- 数据持久化数据库切换为 MySQL

因此当前更适合做“认证 + 发帖 + 浏览文章”的简单联调验证，而不是完整博客社区验收。

## 10. 常见问题

### 10.1 前端端口变化

如果前端不是 `5173`，通常是因为该端口已被占用。

解决方式：
- 直接使用 Vite 启动日志里给出的地址访问

### 10.2 后端数据重置

当前使用 H2 内存数据库。

这意味着：
- 每次重启后端，之前注册的用户和发布的文章都会消失

这是当前开发阶段的正常现象。

### 10.3 前后端无法联调

请确认：
- 后端已经启动在 `8080`
- 前端已经启动
- 前端页面请求的接口地址与后端一致
- 当前允许的本地前端来源地址已配置在后端 `application.yml` 中

## 11. 结论

当前项目已经具备基础运行能力，适合继续在本地推进模块开发与逐步联调。后续随着评论、点赞、后台管理等功能接入，运行方式仍将保持前后端分离结构不变。
