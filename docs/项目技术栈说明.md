# 项目技术栈说明

## 1. 项目概览

Simple Blog 是一个前后端分离的社区博客项目，当前采用：
- 前端：React + TypeScript + Vite
- 后端：Java + Spring Boot + Maven
- 数据库：开发阶段使用 H2，后续可切换到 MySQL

项目目标是逐步实现博客社区常见能力，包括用户登录、发帖、评论、点赞、标签与后台管理。

## 2. 前端技术栈

### 2.1 核心语言与框架

- **语言**：TypeScript
- **UI 框架**：React 19
- **构建工具**：Vite 6

说明：
- TypeScript 用于约束页面组件、路由、状态和接口数据结构
- React 负责页面组件化开发
- Vite 负责开发环境启动、热更新和生产构建

### 2.2 路由与状态

- **路由**：React Router DOM 7
- **状态管理**：Zustand

说明：
- React Router 用于组织首页、登录、注册、发帖、后台等页面路由
- Zustand 当前已安装，后续可用于管理登录态、文章状态和界面状态

### 2.3 HTTP 与样式

- **HTTP 客户端**：Axios
- **样式方案**：原生 CSS

说明：
- Axios 用于后续对接后端 REST API
- 当前样式先采用原生 CSS，方便项目早期快速搭建页面骨架

### 2.4 前端关键配置与入口文件

- `frontend/package.json`
- `frontend/tsconfig.json`
- `frontend/tsconfig.node.json`
- `frontend/vite.config.ts`
- `frontend/index.html`
- `frontend/src/main.tsx`
- `frontend/src/App.tsx`
- `frontend/src/router/AppRouter.tsx`
- `frontend/src/types.ts`

### 2.5 当前前端页面骨架

当前已建立以下页面骨架：
- 首页：`frontend/src/pages/HomePage.tsx`
- 登录页：`frontend/src/pages/LoginPage.tsx`
- 注册页：`frontend/src/pages/RegisterPage.tsx`
- 发帖页：`frontend/src/pages/WritePostPage.tsx`
- 后台页：`frontend/src/pages/AdminPage.tsx`

## 3. 后端技术栈

### 3.1 核心语言与框架

- **语言**：Java 21
- **后端框架**：Spring Boot 3.5.0
- **构建工具**：Maven 3

说明：
- Java 21 作为当前项目运行环境
- Spring Boot 用于快速构建 REST API 与后端业务模块
- Maven 用于依赖管理、测试执行、打包与启动

### 3.2 后端核心依赖

当前后端已接入以下核心依赖：
- **Spring Web**：提供 REST API 能力
- **Spring Data JPA**：提供实体映射与数据访问能力
- **Spring Security**：为登录鉴权和角色权限控制做准备
- **Spring Validation**：用于请求参数校验
- **Flyway**：用于数据库迁移管理
- **JJWT**：用于 JWT 令牌生成与解析
- **Lombok**：用于减少样板代码

### 3.3 测试依赖

- **spring-boot-starter-test**
- **spring-security-test**

说明：
- 前者用于常规单元测试和 Spring 测试支持
- 后者用于安全相关接口和权限测试

### 3.4 后端关键配置与入口文件

- `backend/pom.xml`
- `backend/src/main/java/com/simpleblog/backend/SimpleBlogApplication.java`
- `backend/src/main/resources/application.yml`
- `backend/src/main/resources/db/migration/V1__init.sql`

## 4. 数据库与数据层

### 4.1 当前开发数据库

- **H2 内存数据库**

说明：
- 当前用于本地快速启动与开发验证
- 配置位于 `backend/src/main/resources/application.yml`

### 4.2 目标数据库

- **MySQL**

说明：
- 当前 Maven 依赖中已经预留 MySQL 驱动
- 后续切换生产或持久化开发环境时，可将数据源改为 MySQL

### 4.3 数据库迁移工具

- **Flyway**

说明：
- 使用 SQL 脚本管理表结构演进
- 当前初始化脚本：`backend/src/main/resources/db/migration/V1__init.sql`

### 4.4 当前数据库表

当前初始化脚本已经建立以下表：
- `users`
- `posts`
- `comments`
- `tags`
- `post_tags`
- `post_likes`

这些表用于支撑首版的用户、文章、评论、标签与点赞能力。

## 7. 当前运行方式

### 7.1 前端运行

开发模式：
```bash
npm run dev --prefix frontend
```

生产构建：
```bash
npm run build --prefix frontend
```

### 7.2 后端运行

启动应用：
```bash
mvn -f backend/pom.xml spring-boot:run
```

执行测试：
```bash
mvn -f backend/pom.xml test
```

打包构建：
```bash
mvn -f backend/pom.xml package
```
